Models Configuration¶

Overview¶

Experiment 3 implements the data models for a flight booking system using Django models. The system includes flights, passengers, bookings, and comments.

Model Classes¶

Flight Model¶

from django.db import models

class Flight(models.Model):
    flight_number = models.CharField(max_length=50)
    airline = models.CharField(max_length=100)
    departure_time = models.DateTimeField()
    arrival_time = models.DateTimeField()
    FLIGHT_TYPES = (
        ('ARR', 'Arrival'),
        ('DEP', 'Departure')
    )
    flight_type = models.CharField(max_length=3, choices=FLIGHT_TYPES)
    gate_number = models.CharField(max_length=10)

    def __str__(self):
        return f"{self.flight_number}"

Passenger Model¶

class Passenger(models.Model):
    username = models.CharField(max_length=255, unique=True)
    password = models.CharField(max_length=255)
    bookings = models.ManyToManyField(Flight, through='Booking')

    def __str__(self):
        return self.username

Booking Model¶

class Booking(models.Model):
    passenger = models.ForeignKey(Passenger, on_delete=models.CASCADE)
    flight = models.ForeignKey(Flight, on_delete=models.CASCADE)
    seat_number = models.CharField(max_length=10)

    def __str__(self):
        return f"{self.passenger.username} - {self.flight.flight_number}"

Comment Model¶

class Comment(models.Model):
    flight = models.ForeignKey(Flight, on_delete=models.CASCADE)
    comment_text = models.TextField()
    rating = models.IntegerField(choices=[(i, i) for i in range(1, 11)])
    commenter_name = models.CharField(max_length=100)
    comment_date = models.DateField(auto_now_add=True)

    def __str__(self):
        return f"{self.flight.flight_number} - {self.commenter_name}"

Model Relationships¶

  1. Flight - Passenger (Many-to-Many through Booking)
  2. Passengers can book multiple flights
  3. Flights can have multiple passengers

  4. Booking model tracks seat assignments

  5. Flight - Comment (One-to-Many)

  6. Each flight can have multiple comments
  7. Each comment belongs to one flight

Field Types¶

  1. Text Fields
  2. CharField: For short text (flight numbers, names)

  3. TextField: For longer text (comments)

  4. Date/Time Fields

  5. DateTimeField: For flight times

  6. DateField: For comment dates

  7. Relationship Fields

  8. ForeignKey: For one-to-many relationships

  9. ManyToManyField: For many-to-many relationships

  10. Choice Fields

  11. Flight type choices (ARR/DEP)
  12. Rating choices (1-10)
